# Clean Directory Prompt

散らかったファイル、一時的なスクリプト、確認用テストなどを整理し、プロジェクト構造を改善する。

## Requirements

### 1. ファイル分類基準

#### 削除候補

- `.old` サフィックスのついた旧バージョンファイル
- `temp-`, `tmp-` プレフィックスの一時ファイル
- 空または内容の薄い README
- 使われていないテストファイル

#### 移動・整理候補

- ルートに散らばったドキュメント → `docs/`
- デバッグ用スクリプト → `scripts/debug/`
- 一時的な分析レポート → `docs/reports/`
- 仕様書 → `docs/specs/`

#### 統合候補

- 重複したディレクトリ（例: `specs/` と `docs/specs/`）
- 類似目的のファイル群

### 2. 整理手順

1. 現状把握: list_dir でディレクトリ構造を確認
2. ファイル内容確認: 各ファイルの目的・使用状況を把握
3. ユーザー確認: 削除前に必ず確認を取る
4. 実行: 移動・削除を実施
5. README 更新: 必要に応じて各ディレクトリの README を更新

### 3. ディレクトリ構造の推奨パターン

    project/
    ├── docs/                    # ドキュメント
    │   ├── guides/              # ガイド・チュートリアル
    │   ├── specs/               # 仕様書
    │   └── reports/             # 分析レポート等
    ├── scripts/                 # スクリプト
    │   ├── debug/               # デバッグ用（開発時のみ）
    │   └── migrations/          # マイグレーション用
    ├── tests/                   # テスト
    │   └── fixtures/            # テスト用データ
    └── src/                     # ソースコード

## Constraints

- 削除前確認必須: ユーザーに確認なく削除しない
- git 追跡ファイル注意: .gitignore に含まれないファイルは慎重に
- 依存関係確認: import や require で参照されていないか確認
- バックアップ推奨: 大規模変更前はコミット推奨

## Success Criteria

- 不要ファイルが削除または整理されている
- ディレクトリ構造が論理的に整理されている
- 各ディレクトリに README.md が存在（必要に応じて）
- 重複構造が統合されている
- ファイルの目的が明確に分類されている
